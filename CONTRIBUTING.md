## 1. Code of Conduct

All contributors are expected to engage respectfully and professionally. We are committed to providing a welcoming environment for everyone regardless of background or experience level. Harassment, discrimination, or hostile behavior of any kind will not be tolerated.

If you observe unacceptable behavior, please report it to `paul@emotional-ai-architecture.dev`.

---

## 2. How to Get Help

- **General questions:** Open a [GitHub Discussion](https://github.com/[maintainer]/emotional-ai-architecture/discussions)
- **Bug reports:** Open a [GitHub Issue](https://github.com/[maintainer]/emotional-ai-architecture/issues) using the Bug Report template
- **Feature proposals:** Follow the [RFC Process](#4-requesting-features-rfc-process)
- **Direct contact:** `paul@emotional-ai-architecture.dev`

Before opening an issue, please search existing issues and discussions to avoid duplicates.

---

## 3. Reporting Bugs

Use the **Bug Report** issue template on GitHub. A good bug report includes:

- **Environment:** OS, Python version, relevant dependency versions
- **Module affected:** Which of the 12 modules exhibits the problem (e.g., `modules/affect-engine`)
- **Steps to reproduce:** Minimal, exact steps that reliably trigger the issue
- **Expected behavior:** What you expected to happen
- **Actual behavior:** What actually happened, including full error messages and stack traces
- **Relevant logs or screenshots:** If applicable
