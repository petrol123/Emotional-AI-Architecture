# Contributing to Emotional-AI-Architecture

Thank you for your interest in contributing! This guide will help you get started.

## Structure

This monorepo contains multiple components organized by function:

- \/core\ - Core AI and engine components
- \/research\ - Research projects and findings
- \/hardware\ - Hardware designs and schematics
- \/docs\ - Documentation
- \/src\ - Source code
- \/archive\ - Archive materials

## Getting Started

1. Clone the repository
2. Navigate to the component you want to work on
3. Read that component's README for specific setup instructions
4. Create a feature branch: \git checkout -b feature/your-feature\

## Development Workflow

1. Create a feature branch from main
2. Make your changes in the relevant component directory
3. Test your changes thoroughly
4. Commit with clear, descriptive messages
5. Push to your fork and create a pull request

## Commit Message Guidelines

Use clear, descriptive commit messages:
- \eat: Add new feature\
- \ix: Resolve issue\
- \docs: Update documentation\
- \
efactor: Improve code structure\

## Pull Request Process

1. Update documentation as needed
2. Add tests for new features
3. Ensure all tests pass
4. Provide a clear description of changes
5. Link related issues

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on the work, not the person
- Help others learn and grow

## Questions?

See the individual component READMEs or open an issue with your question.
